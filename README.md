# Surpri3eForexBot — نسخه‌ی یکپارچه و پاک‌سازی‌شده

این نسخه جایگزین **کامل** پروژه‌ی قبلی‌تان است. تمام فایل‌های این بسته را
جایگزین نسخه‌ی گیت‌هابتان کنید و فایل‌های ذکرشده در بخش «حذف شود» را پاک کنید.

## ساختار فایل‌ها

| فایل | وظیفه |
|---|---|
| `main.py` | قلب ربات - همه‌ی دکمه‌ها، پیام‌ها، و منطق تلگرام |
| `database.py` | تنها منبع داده - کاربران، معاملات، تنظیمات، تنظیمات استراتژی، سفارشی‌سازی دکمه‌ها |
| `admin_tools.py` | متن‌های داشبورد و گزارش پنل ادمین |
| `signals.py` | پل بین ربات و `strategy_registry` |
| `strategy_registry.py` | کشف خودکار استراتژی‌های داخل پوشه‌ی `strategies/` |
| `strategies/surpri3e_zigzag.py` | استراتژی Surpri3e (بر پایه‌ی ZigZag) |
| `market.py` | دریافت قیمت/کندل از Twelve Data + دریافت دیتای تاریخی برای بک‌تست + تشخیص باز/بسته بودن بازار |
| `auto_tracker.py` | چک خودکار TP/SL هر ۶۰ ثانیه در پس‌زمینه |
| `backtest.py` | اجرای بک‌تست روی داده‌ی تاریخی واقعی |
| `languages.py` | متن‌های چهار زبان (فارسی، انگلیسی، روسی، عربی) |
| `settings.py` | تنظیمات اولیه‌ی JSON (تایم‌فریم پیش‌فرض و ...) |
| `requirements.txt` | وابستگی‌های پایتون برای Railway |

## ❌ این فایل‌ها را از ریپازیتوری حذف کنید

این‌ها از یک AI دیگر آمده بودند، با معماری متفاوت (دیتابیس جدا `users.db`،
سیستم تنظیمات متفاوت) و باعث کرش ربات می‌شدند چون با فایل‌های بالا ناسازگار بودند:

- `users.py`
- `admin_panel.py`
- `broadcast.py`
- `channel.py`
- `chart.py`
- `logs.py`
- `report.py`
- `tracker.py`
- `ict_logic.py` (قدیمی - جایگزین شده با `strategies/surpri3e_zigzag.py`)
- `smc_logic.py` (قدیمی - همان‌طور بالا)
- `zigzag_logic.py` (قدیمی - جایگزین شده با `strategies/surpri3e_zigzag.py`)

## ویژگی‌های این نسخه

1. **Surpri3e Strategy** بر پایه‌ی ZigZag، با پارامترهای سخت‌گیرِ قابل‌تنظیم
   (Depth, Deviation, Max Age) از پنل ادمین.
2. **دو حالت معاملاتی:** Fast Scalp (فقط تایم‌فریم ۱ دقیقه) و Standard Mode
   (بقیه‌ی تایم‌فریم‌ها).
3. **RR اختصاصی هر کاربر** - هر کاربر نسبت ریسک/ریوارد خودش را جدا از بقیه دارد.
4. **Cooldown سیگنال** بر اساس طول کندل - از سیگنال‌های متناقض پشت‌سرهم جلوگیری می‌کند.
5. **حلقه‌ی تلاش مجدد واقعی** - تا ۹۰ ثانیه چند بار دیتا می‌گیرد و تحلیل می‌کند
   تا سیگنال معتبر پیدا شود، همراه با پیام لودینگ متحرک.
6. **تشخیص بسته بودن بازار** (آخر هفته) - قبل از تحلیل چک می‌شود.
7. **چک خودکار TP/SL** - هر ۶۰ ثانیه در پس‌زمینه قیمت را با معاملات باز
   مقایسه می‌کند و خودکار نتیجه را ثبت و به کاربر اطلاع می‌دهد.
8. **آمار عملکرد هفتگی/ماهانه** - فقط بر اساس معاملاتی که TP/SL آن‌ها
   (دستی یا خودکار) ثبت شده، نه معاملات هنوز باز.
9. **پنل مدیریت استراتژی** - تغییر پارامترهای سخت‌گیری بدون دست‌زدن به کد.
10. **پنل مدیریت دکمه‌ها** - تغییر اسم یا مخفی‌کردن هر دکمه‌ی اصلی از پنل ادمین.
11. **بک‌تست واقعی** - انتخاب بازه‌ی زمانی (ماه شروع/پایان) و اجرای استراتژی
    روی داده‌ی تاریخی واقعی از Twelve Data، با گزارش وین‌ریت واقعی.

## افزودن استراتژی جدید در آینده

هر وقت یک اسکریپت TradingView (Pine Script) جدید داشتید، آن را برای من
بفرستید تا به پایتون تبدیلش کنم و به‌صورت یک فایل جدید در `strategies/`
اضافه کنم (با فرمت استاندارد `STRATEGY_INFO` + `analyze(df)`). از آن پس،
بدون نیاز به تغییر `main.py`، خودش در دکمه‌های ربات و پنل مدیریت ظاهر می‌شود.

## متغیرهای محیطی مورد نیاز (Railway)

- `BOT_TOKEN` - توکن ربات تلگرام (اجباری)
- `ADMIN_ID` - آیدی عددی ادمین (پیش‌فرض: 816822644)
- `TWELVE_DATA` - کلید API برای Twelve Data (برای دیتای واقعی و بک‌تست)
- `CHANNEL_ID` - (اختیاری) آیدی کانالی که برای قفل کانال استفاده می‌شود
- `BOT_USERNAME` - (اختیاری) یوزرنیم ربات برای لینک رفرال
